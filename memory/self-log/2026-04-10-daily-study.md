## 【问题案例1】택배사 지연으로 판매자 점수가 0점이 된 경우

**场景描述**
40대 여성 창업자 셀러가 쿠팡에서 판매 중이었는데, 항상 100점으로 관리하던 판매자 점수가 어느 날 갑자기 배송 부분에서 0점이 되었습니다. 확인해보니 배송이 예상일보다 하루 늦어진 건이 1케이스 있었는데, 실제로는 판매자가 정시 배송을 했고 택배사에서 배송이 지연된 것이었습니다.

**问题分析**
- 쿠팡은 배송 예상일 기준으로 판매자 점수를 산정하여, 택배사의 지연도 판매자에게 패널티가 부과됨
- 판매자가 배송은 정상적으로 했으나 시스템상으로는 판매자 귀책으로 인식됨
- 쿠팡은 소명 요청 시스템이 있으나, 초기에 자동으로 판매자 점수를 차감하는 구조적 문제가 있음

**解决方案**
1. 쿠팡윙(WING)에 로그인하여 해당 배송건의 주문번호와 박스ID(묶음배송번호) 확인
2. 발송 시간 캡쳐 등 증빙 자료 준비
3. 다음 경로로 판매자 점수 제외 요청 접수:  
   `https://helpseller.coupang.com/hc/ko/requests/new?ticket_form_id=360000545712`
4. 양식 작성 시 주문번호, 박스ID, 정시발송완료 사실 명시

**关键要点**
- ⚠️ 택배사 선택 시 배송 속도와 안정성을 반드시 고려해야 함
- 📸 발송 시간 등 증빙 자료는 항상 보관해두는 습관 필요
- 🔄 소명 요청은 가능하나, 처리에 시간이 소요되므로 예방이 최선

**原文链接**: https://nomadel.tistory.com/entry/쿠팡-정시발송했는데-판매자-점수가-갑자기-0이-되었다-판매자-점수-복구-후기

## 【问题案例2】단순 변심 반품을 판매자 귀책으로 처리당한 경우

**场景描述**
위탁판매 셀러가 쿠팡에서 상품을 판매했는데, 구매자가 단순 변심으로 반품을 신청하면서 귀책사유를 '판매자'로 선택했습니다. 이 경우 쿠팡은 반품 배송비를 판매자에게 지급하지 않아 판매자가 왕복 배송비를 모두 부담해야 하는 상황이 발생했습니다.

**问题分析**
- 쿠팡 반품 시스템에서는 반품 배송비 귀책사유를 구매자가 직접 선택
- 일부 구매자가 이를 악용하여 단순 변심임에도 '판매자' 귀책으로 선택하는 경우 발생
- 판매자 귀책으로 처리되면 쿠팡은 반품 배송비를 차감하지 않고 지급도 하지 않음

**解决方案**
1. **구매자와 직접 협의** (가장 추천)
   - 안심번호로 고객에게 연락하여 반품 규정 설명
   - 귀책사유를 '고객(단순변심)'으로 변경 요청 또는 반품 배송비 직접 송금 요청
   
2. **쿠팡 확인 요청 시스템 활용** (협의 실패 시)
   - 입고 후: WING > 반품 관리 > 쿠팡 확인 요청
   - 입고 전: WING > 온라인 문의 > 확인 요청 접수하기
   - 최소 2영업일 이상 구매자와 연락 시도한 증빙(채팅 내역, 통화 기록) 첨부 필수
   
3. **상품 회수 후 168시간(7일) 안에 확인 요청 접수 필요**

**关键要点**
- ⏰ 반품 접수 즉시 대응하는 것이 핵심 (시간 지체 시 권리 상실 가능)
- 📞 고객과의 모든 소통 내역은 반드시 기록/보관
- 🛡️ 사전에 상품 상세페이지에 반품 규정을 명확히 고지하는 것이 예방책

**原文链接**: https://roadwind.tistory.com/entry/쿠팡-반품-처리-방법-feat-단순-변심

## 【问题案例3】반품 후 훼손된 상품에 대한 보상 요청

**场景描述**
셀러가 반품된 상품을 회수했는데, 고객이 사용 흔적이 있거나 훼손된 상태로 반납한 경우입니다. 쿠팡은 이미 고객에게 선환불을 처리했으므로 셀러는 재판매 불가 상품을 그냥 손해로 감수해야 하는 상황입니다.

**问题分析**
- 쿠팡은 고객이 반품 신청 시 선환불이 먼저 이루어지는 경우가 많음
- 회수된 상품이 훼손되었거나 사용 흔적이 있으면 재판매 불가
- 특히 중국산 OEM 제품이나 도매 사이트 상품은 서류를 받기 어려워 보상이 사실상 불가능한 경우도 있음

**解决方案**
1. **제조사로부터 '재판매 불가 확인서' 획득**
   - 제조사/공급처에 연락하여 상품 훼손 확인서 발급 요청
   - 관련 증빙(사진, 검수 보고서 등) 첨부

2. **쿠팡에 확인 요청 접수**
   - 확인서와 함께 쿠팡에 보상 요청
   - 일반적으로 제품가의 약 10% 수준까지 보상 가능

3. **상품 설명에 반품 조건 명확히 고지** (예방책)
   - 상세페이지에 '단순 변심 반품 시 상품 훼손 여부에 따라 환불이 제한될 수 있음' 등 명시

**关键要点**
- 📋 재판매 불가 확인서는 보상 청구의 핵심 증빙 자료
- 🇨🇳 중국산 OEM/도매 상품은 확인서 발급이 어려워 보상이 어려움 (사전 리스크 고려 필요)
- 📝 반품 정책을 사전에 명확히 고지하여 분쟁 예방

## 【问题案例4】로켓그로스 입고 시 입고 거절당한 경우

**场景描述**
셀러가 쿠팡 로켓그로스(물류 대행) 서비스를 이용하기 위해 상품을 입고시켰으나, 물류센터에서 입고 거절을 당했습니다. 바코드 미부착, 포장 규격 미준수, 유통기한 문제 등이 원인이었습니다.

**问题分析**
- 입고 거절 시 반송비(건당 2,000원+)는 셀러가 부담
- 대표적인 입고 거절 사유: 바코드 미부착/불량, 포장 규격 미준수, 유통기한 잔여 70% 미만, 상품 수량 불일치, 라벨 정보 오류
- 로켓그로스는 쿠팡의 풀필먼트 서비스로 엄격한 입고 기준 존재

**解决方案**
1. **입고 전 체크리스트 준수**
   - 표준바코드 또는 쿠팡바코드 정확히 부착 (바코드 없으면 쿠팡바코드 사용 권장)
   - 유통기한은 잔여 70% 이상 유지
   - 포장 규격에 맞는 박스 사용

2. **입고 생성 시 정확한 정보 입력**
   - WING > 로켓그로스 > 입고관리 > 입고생성
   - 상품바코드 인쇄 (폼텍 3102(40칸) 용지 사용)
   - 물류 부착문서와 물류동봉문서 인쇄하여 함께 동봉

3. **입고 거절 시 재시도**
   - 반송된 상품 문제점 수정 후 재입고 신청
   - 반송비는 셀러 부담이므로 첫 입고 시 꼼꼼히 확인하는 것이 중요

**关键要点**
- 🏷️ 바코드는 필수! 표준바코드보다 쿠팡바코드가 사용하기 편리
- 📦 포장 규격 철저히 준수 (5면 부착 권장)
- ⏰ 유통기한 70% 이상 유지 필수

## 【问题案例5】상세페이지를 그대로 등록해서 전환율이 저조한 경우

**场景描述**
많은 초보 셀러들이 공급처(특히 중국 도매처)에서 받은 상세페이지를 한국어 번역 없이 그대로 쿠팡에 등록하는 경우입니다. 중국어로 된 상세페이지는 고객의 불신을 유발하여 클릭률과 전환율이 매우 낮습니다.

**问题分析**
- 중국어 상세페이지는 품질과 상관없이 고객이 불안감을 느낌
- 상세페이지는 고객이 클릭한 후 구매 결정을 내리는 핵심 역할
- 공급처 이미지는 대부분 상품만 덩그러니 있거나 스펙만 나열하여 구매 유도가 약함

**解决方案**
1. **한국어 번역 필수**
   - 공급처 이미지의 중국어를 한국어로 번역
   - 요즘 AI 번역 툴(딥L, 파파고 등)을 활용하면 비용 부담 없이 가능

2. **첫 장에 '후킹 문구' 추가**
   - 상품의 핵심 가치를 전달하는 한 줄 문구 삽입
   - 예: "이 제품 하나로 주방 청소 시간 반으로 줄이세요!"
   - 간결하고 가독성 높은 문구 사용

3. **구매의 긴급성 강조**
   - "누가(Who), 왜(Why) 이 상품을 사야 하는가?"에 대한 답 제시
   - "지금 당장 구매해야 하는 이유"를 구체적인 상황과 함께 제시
   - 예: "오늘 주문 시 내일 출고! 재고 소진 임박"

4. **모바일 최적화**
   - 쿠팡 고객 82% 이상이 모바일 접속
   - 글씨 크기 18~24포인트, 이미지 가로 700픽셀 이상 사용
   - 텍스트보다 인포그래픽 이미지 활용

**关键要点**
- 🇰🇷 한국어 번환만으로도 고객 안심도 크게 상승
- 📱 모바일 환경을 고려한 디자인 필수
- ✨ 후킹 문구는 짧고 간결하게, 핵심만 담기
- 🎯 구매의 긴급성과 구체적인 사용 상황 제시

## 【问题案例6】광고 예산을 낮은 효율로 설정하여 노출이 안 되는 경우

**场景描述**
초보 셀러가 쿠팡 광고(매출 최적화)를 설정할 때 예산을 1만원, 광고 수익률(ROAS)을 1,000%로 설정했더니 광고가 정상적으로 노출되지 않는 상황입니다.

**问题分析**
- 매출 최적화 광고는 ROAS 기준치에 맞춰 광고 노출이 결정됨
- 너무 낮은 예산 + 너무 높은 효율 설정 시 광고가 거의 노출되지 않음
- 비검색 영역의 입찰가를 조절할 수 없어 효율이 떨어질 경우 대응 불가

**解决方案**
1. **초기 설정 (1~2주차)**
   - 일예산: 3~5만원
   - 광고 수익률: 300% 이하로 설정
   - AI 자동 광고로 반응 좋은 키워드 탐색

2. **수동 전환 (3~4주차 이후)**
   - AI로 수집한 데이터 기반으로 수동 광고로 전환
   - 전환율 5% 이하 키워드 제외
   - 성과 좋은 키워드에 예산 집중

3. **브랜드 광고 활용**
   - 비검색 영역 효율이 낮은 상품의 경우 브랜드 광고(CPM 방식)로 전환
   - 키워드로만 광고 집행 가능

4. **보고서 분석**
   - 주기적으로 보고서 출력
   - 클릭률 낮음 → 썸네일/가격/상품명 변경
   - 전환율 낮음 → 상세페이지/리뷰 개선

**关键要点**
- 💰 처음부터 큰 예산보다는 데이터를 쌓으면서 검증된 키워드에 점진적으로 증액
- 📊 보고서 분석을 통한 지속적인 최적화 필수
- 🔄 AI 자동 광고 → 수동 광고 전환 전략 권장

## 【运营技巧】로켓그로스 재고 관리로 보관료 폭탄 피하기

**适用场景**
- 로켓그로스(쿠팡 물류 대행) 서비스를 이용하는 셀러
- 보관료 부담을 줄이고 싶은 셀러
- 재고 회전율을 높이고 싶은 셀러

**具体操作**
1. **무료 보관 기간 활용**
   - 악세서리/의류/신발: 입고 시 45일 무료 (2025.4.1 이후)
   - 기타 카테고리: 입고 시 30일 무료
   - 2025년 3월 31일까지는 모든 카테고리 60일 무료 프로모션 적용

2. **30일 재고 회전 전략**
   - 30일 이내 판매 가능한 재고만 입고
   - 재고관리 자동화 시트 활용 (입고시기, 입고수량 자동 계산)

3. **카테고리별 무료 보관 기간 숙지**
   - 카테고리별로 무료 보관 기간이 다르므로 입고 계획 수립 시 반영

4. **반품/반출 물량 관리**
   - 월 20개까지 무상 처리 가능 (반품 회수비, 반출비)
   - 초과 시 비용 발생하므로 월별 반품량 모니터링

**效果预期**
- 보관료 지출 최소화 (장기 보관 시 보관료가 2배로 증가)
- 재고 회전율 향상으로 현금 흐름 개선
- 무료 보관 기간 내 판매로 순수익 증가

**注意事项**
- ⚠️ 무료 보관 기간 이후에는 일일 단위로 보관료 발생 (10~50원/일)
- ⚠️ 90일 이상 장기 보관 시 보관료 2배 적용
- ⚠️ 입고 전 판매 예측을 정확히 해야 함 (입고 후 반출 시 반출비 발생)

## 【今日总结】

### 核心洞察

1. **반품 귀책사유 관리가 매출과 직결**  
   쿠팡은 단순 변심 반품이라도 고객이 '판매자' 귀책으로 선택하면 반품 배송비를 판매자가 부담해야 합니다. 고객과의 초기 협의가 가장 중요하며, 협의 실패 시 '쿠팡 확인 요청' 시스템을 활용해야 합니다.

2. **로켓그로스 보관료 정책 변화로 재고 관리 전략 필수**  
   2025년 1월부터 무료 보관 기간이 60일에서 30일(일부 카테고리 45일)로 단축되었습니다. 30일 재고 회전 전략을 세우고, 무료 보관 기간 내 판매를 목표로 재고를 관리해야 합니다.

3. **상세페이지 최적화만으로 전환율 30% 이상 상승 가능**  
   공급처 이미지를 그대로 사용하는 대신 한국어 번역, 후킹 문구 추가, 구매 긴급성 강조 등을 통해 상세페이지 품질을 높이면 클릭률과 전환율이 크게 향상됩니다.

### 实操建议

- **내일 바로 적용할 수 있는 것들:**
  1. 현재 등록된 상품의 상세페이지를 점검하여 중국어/영어 부분 한국어로 번역
  2. 반품 처리 프로세스 정리하고 '쿠팡 확인 요청' 접수 경로 bookmark
  3. 로켓그로스 이용 시 재고관리 시트를 만들어 30일 이내 판매 가능량만 입고하도록 계획
  4. 광고 설정이 너무 보수적(낮은 예산+높은 ROAS)이라면 초기에 적절히 예산 확보

- **단기 전략 (1주일 내):**
  1. 상세페이지 첫 장에 후킹 문구 추가하여 A/B 테스트
  2. 택배사별 배송 속도 데이터를 수집하여 안정적인 택배사로 변경 검토
  3. 반품 귀책사유 변경 관련 고객 응대 템플릿 작성
